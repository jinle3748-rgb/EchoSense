# 人声分析功能使用说明

## 📖 功能概述

人声分析标签页集成了完整的声学特征提取和情绪分析功能，可以对音频进行多维度分析。

## 🎯 主要功能

### 1. 声学特征提取
- **基频(F0)特征**：平均基频、基频范围、基频标准差
- **频谱特征**：频谱质心、频谱带宽
- **能量特征**：RMS能量、能量动态范围
- **语音质量**：HNR、Jitter、Shimmer

### 2. 情绪分析
支持7种基本情绪识别：
- 中性（Neutral）
- 快乐（Happy）
- 悲伤（Sad）
- 愤怒（Angry）
- 恐惧（Fear）
- 惊讶（Surprise）
- 厌恶（Disgust）

### 3. 并行计算优化
- 支持多进程并行处理
- 自动性能对比
- 加速比计算

## 📝 使用步骤

### 1. 加载音频文件
- 切换到"音频导入"标签页
- 拖拽音频文件或点击"选择文件"
- 支持格式：.wav, .mp3, .flac, .ogg

### 2. 进行人声分析
- 切换到"人声分析"标签页
- 选择分析模式：
  - **完整分析**：提取声学特征 + 情绪分析
  - **仅声学特征**：只提取声学特征
  - **仅情绪分析**：只进行情绪分析
- 勾选"并行处理"启用多进程加速（推荐）
- 点击"开始分析"按钮

### 3. 查看分析结果
分析完成后会显示：
- **音频信息**：文件路径、时长、采样率、分析段数
- **声学特征**：各项声学参数的具体数值
- **情绪分析**：
  - 主导情绪及置信度
  - 情绪强度
  - 次要情绪
- **情绪分布**：各情绪在时间线上的占比
- **性能信息**：分析耗时

### 4. 可视化展示
右侧画布会显示：
- 情绪分布条形图
- 主导情绪及置信度信息

### 5. 导出结果
- 点击"导出结果"按钮
- 选择保存路径
- 导出为JSON格式

## ⚙️ 参数说明

### 分析模式
- **完整分析**：推荐使用，提取所有特征
- **仅声学特征**：不需要情绪分析时使用，速度更快
- **仅情绪分析**：不需要声学特征时使用

### 并行处理
- 启用：使用多进程并行处理，速度更快
- 禁用：使用串行处理，便于调试

## 🎨 情绪识别原理

情绪识别基于以下声学特征：

| 情绪 | 主要特征 |
|------|----------|
| 快乐 | 高基频、高能量、高频谱质心 |
| 悲伤 | 低基频、低能量、低频谱质心 |
| 愤怒 | 高基频、高能量、不稳定 |
| 恐惧 | 高基频、高变化、能量波动大 |
| 惊讶 | 高基频、大基频范围、高能量 |
| 厌恶 | 低基频、低能量、低频谱质心 |
| 中性 | 中等基频、中等能量 |

## 💡 注意事项

1. **音频质量**：清晰、无噪音的音频分析效果更好
2. **语音内容**：单人语音分析效果优于多人对话
3. **音频长度**：建议1秒以上，太短可能无法准确分析
4. **采样率**：支持各种采样率，建议16kHz以上
5. **性能**：并行处理可显著提升分析速度

## 🔧 高级功能

### 批量分析
在代码中可以调用`analyze_batch()`方法进行批量分析：
```python
from VoiceAnalysis import VoiceAnalyzer

analyzer = VoiceAnalyzer(use_parallel=True)
results = analyzer.analyze_batch(audio_files)
```

### 性能对比
使用`compare_parallel_performance()`对比并行和串行性能：
```python
performance = analyzer.compare_parallel_performance(test_file)
print(f"加速比: {performance['speedup']:.2f}x")
```

## 📞 故障排除

### 分析失败
- 检查音频文件是否损坏
- 确保音频格式受支持
- 检查内存是否充足

### 性能问题
- 启用并行处理提升速度
- 减小分析窗口大小
- 使用GPU加速（如可用）

### 结果不准确
- 确保音频为单人语音
- 音频不应包含过多背景噪音
- 避免分析过短的音频片段
